O que é o Projeto D.I.A?

Acessibilidade não é uma feature, é a base. O Projeto D.I.A. nasceu da necessidade de criar um recurso centralizado, em português, que transforma as complexas diretrizes da WCAG em um fluxo de trabalho prático e aplicável para o dia a dia de designers e desenvolvedores. Acreditamos que a tecnologia só cumpre seu propósito quando funciona para todos.

OS 4 Pilares da Acessibilidade

Perceptível

A informação e os componentes da interface devem ser apresentados de forma que os usuários possam percebê-los, usando diferentes sentidos.

Operável

Os componentes da interface e a navegação devem ser operáveis. O usuário deve conseguir interagir com todos os controles.

Compreensível

A informação e a operação da interface devem ser compreensíveis. O conteúdo deve ser claro e o sistema previsível.

Robusto

O conteúdo deve ser robusto o suficiente para ser interpretado de forma confiável por uma ampla variedade de agentes de usuário, incluindo tecnologias assistivas.
